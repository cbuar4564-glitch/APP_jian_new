# 안전 기준 (우리 앱 사업의 리스크 규칙)

> 목적: 새 앱을 만들기 전에 "이거 위험한가?"를 매번 고민하지 않도록, **한 번 정해두고 계속 적용하는 기준**.
> 원칙: **우리가 틀려도 사용자가 돌이킬 수 없는 큰 손해(돈·법·건강)를 입지 않는 앱만 만든다.**

---

## 1. 우리가 질 수 있는 위험 7가지

| # | 위험 | 무슨 일이 생기나 |
|---|------|------------------|
| 1 | **법적 책임** | 잘못된 정보·판단으로 사용자가 손해 → 손해배상 청구 (특히 법률·금융·의료·안전) |
| 2 | **개인정보 유출** | 사용자 데이터를 보관하다 유출 → 과징금(개인정보보호법)·신뢰 붕괴 |
| 3 | **금전 사고** | 결제·정산·환불을 직접 다루다 분쟁·사기·차지백 |
| 4 | **무자격 영업** | 금융·의료·세무·법무·부동산중개는 자격/인허가 필요 → 무자격 시 위법 |
| 5 | **콘텐츠 책임** | 부적절·유해·저작권 침해 콘텐츠 생성/노출 |
| 6 | **운영 실패** | 서버 다운·버그로 서비스 실패 (저위험이나 신뢰 손상) |
| 7 | **세무 의무** | 수익 발생 시 사업자등록·세금 신고 의무 |

---

## 2. 우리의 기본 방어막 (이미 가진 강점)

이 4가지를 지키면 위 위험의 대부분이 자동으로 막힌다.

1. **기기 내 처리** → 데이터를 우리가 안 가짐 → 위험 ②(유출) 거의 제거
2. **돈을 직접 안 다룸** (받더라도 검증된 외부 PG에 위임) → 위험 ③ 완화
3. **"정보 제공·참고용"만, 단정 안 함** → 위험 ①⑤ 완화
4. **"대행·자문"이 아니라 "도구·작성 보조"** → 위험 ④(무자격) 회피

---

## 3. 신호등 분류 — 새 앱은 여기에 먼저 넣어본다

### 🟢 GREEN — 만들어도 됨 (저위험)
- **창작·표현 도구**: 이야기·메시지·카드·디자인·홍보물 → 틀려도 사용자가 보고 고침
- **단순 계산기**(참고용 명시, 사용자가 입력·검증), 정리·변환·생산성 유틸
- **정보 정리·체크리스트**(단정 없이 "확인해 보세요" 수준)
- 조건: 데이터 기기 내 · 결제 직접 안 함 · 면책 명시 · "판단"은 사용자가

### 🟡 YELLOW — 조건부 (안전장치 달면 가능)
- 돈·계약·건강 관련 **정보 제공**이되 단정 위험이 있는 것 (예: 환불 안내, 위약금 *참고* 계산)
- 결제를 **검증된 외부 PG**(토스·Gumroad 등)에 100% 위임하는 단순 판매
- 조건: 강한 면책 + 공식 출처 안내 + "최종 확인은 전문가·공식기관" 명시 + 사업자등록

### 🔴 RED — 안 만든다 (고위험)
- **"안전하다 / 합법이다 / 이 금액이 맞다"고 단정하는 진단·판단**
  (전세 안전진단, 중고차 사고·침수 판정, 세액 확정, 근로계약 합법성 판정 등)
- 무자격으로 **금융·의료·세무·법무·부동산중개** 자문·대행
- 사용자 **민감정보(주민번호·계좌·건강기록)를 서버에 저장**
- 우리가 **돈을 직접 보관·정산·송금**
- **의료·안전·법적 결과**를 좌우하는 조언

---

## 4. 모든 앱 필수 안전장치 (출시 전 체크리스트)

- [ ] 데이터는 **기기 안에서만** 처리 (서버 저장 0)
- [ ] 결제를 **직접 받지 않음** (받으면 검증된 외부 PG + 사업자등록)
- [ ] **"참고용·정보 제공"** 면책 문구 명시, 법률·금융·의료 **단정 금지**
- [ ] **"판단"을 우리가 하지 않음** — 사용자가 결과를 보고 직접 결정
- [ ] 개인정보·비밀파일 커밋 차단(.gitignore)
- [ ] **이용약관·면책조항** 페이지 (수익화 시점)
- [ ] 외부 전송 없음 (또는 명시·동의)
- [ ] 생성 콘텐츠가 유해·저작권 침해 소지 없는지 점검

---

## 5. Go / No-Go — 한 줄 테스트

> **"이 앱이 *틀린 결과*를 냈을 때, 사용자가 *돌이킬 수 없는 큰 손해*(돈·법·건강)를 입는가?"**

- **예** → 🔴 안 만든다 (또는 YELLOW 안전장치 필수)
- **아니오** (보고 다시 하면 됨) → 🟢 진행

---

## 6. 기존 앱 점검 결과

| 앱 | 분류 | 사유 |
|---|:--:|---|
| 🌙 꿈동산 | 🟢 GREEN | 창작 콘텐츠. 틀려도 무해. |
| 💰 내돈지킴이 | 🟡 YELLOW (장치 적용됨) | 돈·법 인접하나 "단정 안 함 + 참고용 + 기기내 + 결제 안 받음"으로 위험 차단. 현 상태 유지 OK. |

> 앞으로 모든 신규 앱은 **3번 신호등 → 5번 한 줄 테스트 → 4번 체크리스트** 순으로 통과해야 만든다.
