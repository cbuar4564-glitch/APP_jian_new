## 목표

이 프로젝트는 사람들이 생활 속에서 겪는 불편함을 AI로 해결하는 앱을 만든다.

우선 해결할 문제는 다음 3가지다.

1. 내돈지킴이
- 구독 해지
- 환불
- 가격 속임수
- 계약금/위약금
- 소비자 신고 문구 작성

2. 아이 잠자리 루틴
- 아이 재우기
- 부모 피로
- 감정형 잠자리 이야기
- 수면 루틴 관리

3. 1인가구 생활안심
- 혼자 사는 사람의 안부 확인
- 약/병원/공과금 알림
- 응급 연락
- 생활관리

## 작업 방식

바로 코딩하지 말고 다음 순서로 진행한다.

1. 문제 정의
2. 사용자 정리
3. 핵심 기능 3개 선정
4. 화면 구성
5. MVP 제작
6. 테스트
7. 개선

각 단계가 끝나면 내가 이해하기 쉽게 요약 보고한다.

## 중요한 원칙

- 처음부터 복잡한 폴더 구조를 만들지 않는다.
- 필요한 파일과 폴더는 Claude가 판단해서 최소한으로 만든다.
- 파일을 만들기 전에 왜 필요한지 간단히 설명한다.
- MVP를 먼저 만든다.
- 예쁜 기능보다 실제 불편 해결을 우선한다.
- 사용자가 돈을 낼 만한 기능을 우선한다.

## 개인정보 보호

- .env 파일은 절대 읽지 않는다.
- API 키, 토큰, 비밀번호, 개인정보는 절대 출력하지 않는다.
- 실제 사용자 데이터 대신 더미 데이터를 사용한다.
- 개인정보가 들어간 파일은 커밋하지 않는다.
- 외부 API로 민감한 내용을 보내지 않는다.
- 법률, 금융, 의료 판단은 단정하지 않는다.
- 환불, 계약, 보험 관련 기능은 정보 제공 수준으로 만든다.

## Claude의 역할

Claude는 단순 코더가 아니라 기획자, PM, 개발자, QA 역할을 함께 수행한다.

항상 다음을 스스로 판단한다.

- 이 기능이 진짜 불편을 해결하는가?
- 사용자가 돈을 낼 만큼 중요한가?
- 너무 복잡하지 않은가?
- 개인정보 위험은 없는가?
- MVP로 먼저 만들 수 있는가?
